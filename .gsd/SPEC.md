# SPEC.md — Project Specification

> **Status**: `FINALIZED`

## Vision
Migrate the WhatsApp AI chatbox from Google Gemini to Groq (LPU) to improve response speed and leverage high-performance open-source models like Llama 3.3.

## Goals
1. Replace Gemini LLM integration with Groq.
2. Maintain the "Hospital Management" persona and helpfulness.
3. Improve inference latency for WhatsApp users.
4. Ensure robust handling of API keys and environment variables.

## Non-Goals (Out of Scope)
- Adding new features to the WhatsApp bot (beyond LLM migration).
- Changing the WhatsApp API provider (Meta/Twilio/etc).
- Implementing persistent vector storage (RAG).

## Users
Patients and hospital staff interacting with the WhatsApp bot for queries and appointments.

## Constraints
- **Technical:** Must remain a Python FastAPI application.
- **Library:** Use the `groq` Python SDK.
- **Model:** Default to `llama-3.1-8b-instant` on Groq Cloud.

## Success Criteria
- [ ] AI responses are generated by Groq (verified via logs).
- [ ] Hospital system prompt is correctly applied to responses.
- [ ] End-to-end message flow from WhatsApp to Groq and back is functional.
- [ ] System handles Groq API errors (rate limits, context window) gracefully.
